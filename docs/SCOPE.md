# SCOPE.md — Договорённости и рамки проекта (Этап 0)

Зафиксировано в рамках Этапа 0. Меняется только по явному решению, не по ходу дела.

## Окружение разработки

- ОС разработки: Windows 11.
- Python: 3.11 (`C:\Program Files\Python311`). Изначально стоял 3.14.6, но при установке зависимостей (Этап 4) выяснилось, что под 3.14 нет готовых бинарных сборок для `pydantic-core` и `aiohttp` — потребовалась бы компиляция из исходников (Rust + MSVC Build Tools). Перешли на 3.11, где для всего стека есть готовые wheels.
- Виртуальное окружение проекта: `c:\Users\tokar\.virtualenvs\.venv` (пересоздано под Python 3.11).
- В системе настроен локальный SOCKS-прокси (реестр Windows, `127.0.0.1:10808`) — pip пытается через него подключаться и падает с ошибкой SOCKS-поддержки. Обход: перед установкой пакетов выставлять `$env:NO_PROXY = "*"` и `$env:no_proxy = "*"` в PowerShell-сессии.

## Источник данных Steam

- Steam Web API (официальный, по API-ключу) — где применимо.
- Публичный price history эндпоинт Steam Community Market — без API-ключа, но:
  - соблюдаем разумные паузы между запросами (rate limit);
  - без обхода антибот-защиты и без имитации браузера ради обхода лимитов;
  - если для какой-то функции официального/документированного способа получения данных нет — вопрос обсуждается до реализации, а не решается агрессивным скрапингом молча.

## Позиционирование сервиса

- Это аналитический инструмент: сбор истории цен, статистический анализ, сигналы.
- Это не финансовая рекомендация и не лицензированная инвестиционная консультация.
- Во всех пользовательских интерфейсах (Telegram-бот, API, будущий UI) формулировки должны явно давать понять: результат — статистический сигнал, а не финансовый совет.
